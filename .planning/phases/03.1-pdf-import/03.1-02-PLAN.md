<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User browser → /api/analyze-pdf | Multipart file upload — must include Bearer token (enforced by Plan 01) |
| User browser → /api/transactions/import | JSON payload with raw extraction string |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.1-07 | Spoofing | ImportPdfModal — token from localStorage | accept | getToken() reads fincoach_access_token from localStorage, same pattern used everywhere in this app. XSS risk is accepted for MVP (5-10 internal users). |
| T-03.1-08 | Tampering | Crafted result JSON passed to /api/transactions/import | mitigate | Backend validates JSON structure and enforces user_id from JWT (Plan 01). Frontend never constructs result — passes through from analyze-pdf verbatim. |
| T-03.1-09 | Information Disclosure | PDF content visible in browser memory | accept | PDF is read as FormData and sent immediately. Not stored in state. Content only travels to Anthropic via the backend proxy. |
| T-03.1-10 | Denial of Service | User uploads very large PDF repeatedly | accept | MVP has 5-10 users. Anthropic enforces 32MB limit on their end. No client-side size restriction needed for MVP. |
| T-03.1-11 | Elevation of Privilege | Modal submits without auth token | mitigate | apiFetch and the manual fetch both include Authorization header. Backend rejects with 401 if token absent. |
</threat_model>
