<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend → /api/analyze-pdf | Unauthenticated file upload was possible — now requires JWT |
| Frontend → /api/transactions/import | New endpoint accepting JSON payload with transaction data |
| Backend → Supabase | Authenticated client with RLS — user can only insert their own rows |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.1-01 | Elevation of Privilege | /api/analyze-pdf | mitigate | Add get_current_user dependency (per D-05). Without this, any anonymous caller can consume the Anthropic API key. |
| T-03.1-02 | Spoofing | /api/transactions/import user_id | mitigate | user_id always from JWT via get_current_user, never from request body (per D-06). RLS policy `INSERT: auth.uid() = user_id` is second layer. |
| T-03.1-03 | Tampering | Malformed JSON in import payload | mitigate | json.loads wrapped in try/except, raises 422 on invalid JSON. |
| T-03.1-04 | Information Disclosure | /api/analyze-pdf PDF content | accept | PDF is processed in-memory, not persisted. Claude Haiku only receives base64 content. Risk low for MVP. |
| T-03.1-05 | Denial of Service | Large PDF upload | accept | FastAPI default max body size + Anthropic's 32MB PDF limit applies. Rate limiting out of scope for MVP (5-10 users). |
| T-03.1-06 | Repudiation | Bulk insert with no audit log | accept | Supabase has built-in created_at timestamps. Full audit trail is v2 scope. |
</threat_model>
