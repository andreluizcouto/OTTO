# Phase 03.1: PDF Import — Context

## Phase Goal
Users can upload a bank statement PDF directly in the Transactions page, have transactions extracted by AI, and have them saved to their Supabase account automatically.

## Locked Decisions

### PDF password handling
Encrypted PDFs are **not supported** in the upload flow. The UI must show a clear error/warning before upload (e.g., "Seu PDF deve estar sem senha. PDFs criptografados não são suportados."). The `/api/decrypt` endpoint exists but is NOT part of this flow — user must decrypt externally before uploading.

### Save behavior
Transactions are saved **immediately** after extraction — no review step. User corrects individual transactions after the fact via the existing transaction edit/category correction flow.

### Where the save happens
**Backend endpoint `POST /api/transactions/import`** handles the insert. The frontend React app has no supabase-js client installed — all data operations go through FastAPI. This follows the existing project pattern.

The `user_id` is derived from the JWT via `get_current_user` dependency, never accepted from the request payload. RLS on the `transactions` table (`INSERT: auth.uid() = user_id`) enforces this at the database layer as a second check.

### Duplicate detection
Before inserting, check for existing transactions with the same `(date, amount, description)` tuple for the current user. If a match is found:
- Skip the duplicate silently
- Show a summary toast at the end: "X transações importadas, Y já existiam e foram ignoradas."

### Authentication on `/api/analyze-pdf`
Add `get_current_user` dependency to the endpoint. This protects the Anthropic API key from unauthenticated use. The endpoint only extracts — it does not insert anything into the database.

## Data Mapping
The Claude extraction returns:
```json
{"transacoes": [{"data": "DD/MM/AAAA", "descricao": "...", "valor": 0.00, "tipo": "debito|credito"}]}
```

Map to `transactions` table fields:
- `date` ← `data` (parse DD/MM/AAAA → YYYY-MM-DD)
- `description` ← `descricao`
- `amount` ← `valor` as positive for crédito, negative for débito
- `user_id` ← from authenticated Supabase session
- `category_id` ← null (unclassified, will go through existing classify flow)
- `merchant_name` ← null (resolve later via classify)
- `confidence_score` ← null
- `manually_reviewed` ← false

## UI Entry Point
The existing "Importar" button in `Transactions.tsx` triggers this flow. Implementation likely via a modal/dialog.

## Out of Scope
- Two-step decrypt + analyze flow
- Transaction review before save
- AI classification during import (happens separately via existing classify CTA)
- Bulk import history / import log
