## Component Inventory

### KPI Cards (4 cards in a row via st.columns)

| KPI | Icon (Bootstrap) | Label | Format |
|-----|-----------------|-------|--------|
| Total spent | `cash-stack` | Total no mes | R$ X.XXX,XX |
| Transaction count | `receipt` | Transacoes | N transacoes |
| Top category | `trophy` | Categoria mais cara | Emoji + Name |
| Daily average | `calendar3` | Media por dia | R$ X.XXX,XX |

**Card structure (HTML/CSS via st.markdown):**
- Background: #1E293B
- Border: 1px solid #334155
- Border-radius: 8px
- Padding: 24px (lg)
- Icon: 20px, color #94A3B8, displayed above metric
- Metric: 28px Display weight 600, color #F8FAFC
- Label: 14px Label weight 400, color #94A3B8
- Delta indicator (optional): 14px, color #22C55E (down) or #EF4444 (up), shown below metric as "+12%" or "-8% vs mes anterior"

### Filter Bar

- Component: st.selectbox
- Position: top of page, above KPI cards
- Options: "Esta semana" | "Este mes" | "Ultimos 3 meses"
- Default: "Este mes"
- Behavior: changes all KPIs, charts, and table simultaneously (single global filter per D-12)

### Donut Chart (Plotly)

- Type: plotly.graph_objects.Pie with hole=0.55
- Colors: mapped from categories table color_hex
- Center annotation: total amount in R$ format, 20px weight 600, color #F8FAFC
- Legend: outside right, 14px, color #94A3B8
- Hover: category name + R$ amount + percentage
- Background: transparent (inherits page bg)
- Section heading: "Gastos por Categoria" (Heading 20px/600)

### Trend Line Chart (Plotly)

- Type: plotly.graph_objects.Scatter with mode='lines+markers'
- Line color: #2563EB, width 2
- Fill: tozeroy, fillcolor rgba(37, 99, 235, 0.1)
- Markers: 6px circles, #2563EB
- X-axis: dates (week or month labels depending on filter)
- Y-axis: R$ amounts, formatted with Brazilian locale
- Gridlines: #334155 at 0.3 opacity
- Background: transparent
- Section heading: "Tendencia de Gastos" (Heading 20px/600)

### Comparison Bar Chart (Plotly)

- Type: plotly.graph_objects.Bar (grouped, barmode='group')
- Current month bars: #2563EB
- Previous month bars: #475569
- X-axis: category names
- Y-axis: R$ amounts
- Legend: "Mes atual" / "Mes anterior", 14px, color #94A3B8
- Bar gap: 0.2, bargroupgap: 0.1
- Background: transparent
- Section heading: "Atual vs Mes Anterior" (Heading 20px/600)

### Recent Transactions Table

- Component: st.dataframe or custom HTML table
- Rows: last 20 transactions, sorted by date descending
- Columns:
  - Data: formatted as DD/MM (14px, #94A3B8)
  - Merchant: merchant_name (16px, #F8FAFC)
  - Categoria: emoji + category name (16px, category color_hex)
  - Valor: R$ amount right-aligned (16px, #F8FAFC)
- Section heading: "Transacoes Recentes" (Heading 20px/600)
- Stripe background: alternate rows at #1E293B / transparent
- If st.dataframe: hide index, configure column widths

### Settings Page Additions (Data Generation)

- Section heading: "Dados de Teste" (between Conta and Sessao sections)
- Button "Gerar Dados": accent #2563EB background, white text, use_container_width=True
- Help text below button: "Gera 3 meses de transacoes simuladas com merchants brasileiros" (14px, #64748B)
- Button "Limpar Dados": destructive style (border #EF4444, text #EF4444, hover fill #EF4444)
- Success feedback: st.success("X transacoes geradas com sucesso!")
- Warning on existing data: st.warning("Voce ja tem transacoes. Limpe antes de gerar novos dados.")

## Monetary Formatting

All R$ values in the UI must use Brazilian locale:
- Thousands separator: dot (.)
- Decimal separator: comma (,)
- Currency symbol: R$ with non-breaking space before number
- Examples: R$ 4.230,50 | R$ 25,00 | R$ 1.500,00
- Implementation: `f"R$ {value:,.2f}".replace(",", "X").replace(".", ",").replace("X", ".")`

## Interaction States

### Dashboard Page

| State | Trigger | Visual |
|-------|---------|--------|
| Loading | Page load, filter change | st.spinner("Carregando dashboard...") wrapping all content |
| Empty | No transactions in DB | Empty state card (placeholder-card style) with heading + body + link to Settings |
| Empty (filtered) | No transactions in selected period | Inline message below filter: "Nenhuma transacao neste periodo." |
| Populated | Transactions exist | Full dashboard: KPIs -> Donut -> Trend -> Comparison -> Table |
| Error | Supabase query fails | st.error("Erro ao carregar dados. Verifique sua conexao e tente novamente.") |

### Settings Page (Data Generation)

| State | Trigger | Visual |
|-------|---------|--------|
| No data | 0 transactions | "Gerar Dados" button enabled, "Limpar Dados" button disabled |
| Has data | >0 transactions | "Gerar Dados" hidden, show "Limpar Dados" + "Regerar" button pair |
| Generating | Button click | st.spinner("Gerando transacoes...") replacing buttons, button disabled |
| Success | Generation complete | st.success("{N} transacoes geradas com sucesso!"), auto-rerun to refresh |
| Clearing | Clear button after confirmation | st.spinner("Removendo dados...") |
| Cleared | Clear complete | st.success("Dados removidos com sucesso."), show "Gerar Dados" button |
| Error | Operation fails | st.error with specific message |

### Destructive Action Flow (Limpar Dados)

1. User clicks "Limpar Dados"
2. st.warning("Tem certeza? Isso removera todas as suas transacoes.")
3. Two columns: "Confirmar" (destructive red) / "Cancelar" (default)
4. On confirm: delete, show success, rerun
5. On cancel: collapse warning, return to normal state
