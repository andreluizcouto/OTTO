4. **`generate_transactions(user_id: str, category_map: dict, months: int = 3, per_month_range: tuple = (40, 60)) -> list[dict]`:**
   - Import `random`, `from datetime import date, timedelta`
   - For each `month_offset` in `range(months)`:
     - Calculate `month_start` = 1st of month, going back `month_offset` months from today
     - Calculate `month_end` = last day of that month (handle December edge case)
     - If `month_offset == 0`, cap `month_end` at `today` (don't generate future transactions)
     - `n_transactions = random.randint(*per_month_range)`
     - `slugs = list(CATEGORY_WEIGHTS.keys())`
     - `weights = list(CATEGORY_WEIGHTS.values())`
     - `chosen_categories = random.choices(slugs, weights=weights, k=n_transactions)`
     - For each `slug` in `chosen_categories`:
       - `cat = category_map[slug]`
       - `merchant = random.choice(MERCHANTS[slug])`
       - `min_amt, max_amt = AMOUNT_RANGES[slug]`
       - `amount = round(random.uniform(min_amt, max_amt), 2)`
       - Generate weighted day: use triangular distribution `random.triangular(1, month_end.day, mode)` where `mode` is biased toward start (salary) and end (bills). Implement as: pick either start-biased or end-biased with 50/50 chance. Start-biased: `mode=5`. End-biased: `mode=month_end.day - 3`. Clamp to valid range and convert to `int`.
       - `txn_date = month_start.replace(day=max(1, min(day, month_end.day)))`
       - Append dict:
         ```python
         {
             "user_id": user_id,
             "amount": amount,
             "date": txn_date.isoformat(),
             "description": merchant,
             "merchant_name": merchant,
             "category_id": cat["id"],
             "confidence_score": "high",
             "payment_method": random.choice(["credito", "debito", "pix"]),
             "is_recurring": slug in ("assinaturas", "moradia"),
         }
         ```
   - Return the full list of transaction dicts.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Tampering | generator.py | mitigate | All generated transactions use the authenticated user's ID from get_current_user()["id"]; RLS INSERT policy requires auth.uid() = user_id — cannot insert data for another user |
| T-02-04 | Tampering | settings.py delete | mitigate | Delete scoped with .eq("user_id", user["id"]) AND RLS DELETE policy requires auth.uid() = user_id — double protection |
| T-02-05 | Denial of Service | settings.py generate | mitigate | Check txn_count before allowing generation; if >0 show warning instead of button — prevents duplicate bulk inserts |
| T-02-06 | Input Validation | generator.py | mitigate | All generated values constrained: amount within AMOUNT_RANGES, date within 3-month window, payment_method from fixed list, confidence_score hardcoded "high" — matches schema CHECK constraints |
</threat_model>
