## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding within form fields |
| sm | 8px | Compact element spacing, gap between label and input |
| md | 16px | Default element spacing, form field margins |
| lg | 24px | Section padding, card internal padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page top/bottom padding |
| 3xl | 64px | Login card vertical centering offset |

Exceptions: Streamlit manages most spacing internally. Custom spacing applies only to injected CSS blocks (login form layout, sidebar header, placeholder cards). Use `st.markdown` with inline styles or `<style>` blocks referencing these values.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Paragraphs, form labels, table cells, descriptions |
| Label | 14px | 400 (regular) | 1.4 | Secondary text, captions, helper text, timestamps |
| Heading | 24px | 600 (semibold) | 1.2 | Page titles ("Dashboard", "Configuracoes"), section headers |
| Display | 32px | 600 (semibold) | 1.1 | Login page app title "FinCoach AI" only |

**Weights used:** 2 total -- 400 (regular) and 600 (semibold). No other weights.

**Streamlit overrides:** Streamlit controls most text sizing. These values apply to custom-injected elements (login page, placeholder cards, sidebar branding). Use `st.title()` for page headings (maps to Display), `st.header()` for sections (maps to Heading), `st.write()` / `st.markdown()` for body text.

## Component Inventory (Phase 1)

### 1. Login/Signup Page

**Layout:** Centered single column, max-width 400px, vertically centered in viewport. No sidebar visible on this page.

**States:**
| State | What the user sees |
|-------|-------------------|
| Default (login) | Email field + password field + "Entrar" button (blue) + "Criar conta" link below |
| Signup mode | Email field + password field + confirm password field + "Criar conta" button (orange) + "Ja tenho conta" link below |
| Loading | Spinner on button, fields disabled |
| Login error | Red error banner above form: "Email ou senha incorretos. Tente novamente." |
| Signup error (email exists) | Red error banner: "Este email ja esta cadastrado. Tente fazer login." |
| Signup error (weak password) | Red error banner: "A senha deve ter pelo menos 6 caracteres." |
| Signup success | Green success banner: "Conta criada! Verifique seu email para confirmar." (if Supabase email confirmation is enabled) OR redirect to dashboard |

**Interaction:**
- Toggle between login and signup via text link (not tabs)
- Password field uses `type="password"` (Streamlit `st.text_input` with `type="password"`)
- "Entrar" button triggers `client.auth.sign_in_with_password()`
- "Criar conta" button triggers `client.auth.sign_up()`
- On successful login, redirect to Dashboard page with sidebar visible
- On page load, check `st.session_state` for existing session -- if valid, skip login

### 2. Sidebar Navigation

**Component:** `streamlit-option-menu` in `st.sidebar`

**Items (Phase 1):**
| Label | Icon | Active | Destination |
|-------|------|--------|-------------|
| Dashboard | `house` | Yes (default) | Dashboard placeholder page |
| Configuracoes | `gear` | No | Settings page |

**Future items (grayed out or hidden until their phase):**
- Transacoes (Phase 2)
- Orcamentos (Phase 4)
- Metas (Phase 5)

**Sidebar header:** App name "FinCoach AI" in semibold 24px (Heading) with `#2563EB` color, top of sidebar above navigation items.

**Sidebar footer:** User email displayed in `#94A3B8` (text-secondary) at 14px, with "Sair" logout link/button below it.

**Styling overrides** (via custom CSS):
- Active item: `#2563EB` background with `#F8FAFC` text
- Inactive items: transparent background with `#94A3B8` text
- Hover: `#1E293B` background lightened slightly to `#334155` (slate-700)

### 3. Dashboard Placeholder

**Layout:** Single column, full width within Streamlit main area.

**Content:**
- Page heading: `st.title("Dashboard")` -- renders as "Dashboard" in Display size
- Placeholder card: A styled `st.container` or `st.markdown` block with:
  - Background: `#1E293B`
  - Border: 1px solid `#334155` (slate-700)
  - Border-radius: 8px
  - Padding: 24px
  - Content: centered text "Seus dados financeiros aparecerao aqui" in `#94A3B8` at 16px
  - Below text: "Os graficos e resumos serao adicionados na proxima fase." in `#64748B` (slate-500) at 14px

### 4. Settings Page

**Layout:** Single column, max-width 600px.

**Content:**
- Page heading: `st.title("Configuracoes")`
- Section: "Conta" -- displays user email (read-only) and account creation date
- Section: "Sessao" -- "Sair da conta" button (red outline style, not filled)
- Future sections placeholder: A subtle divider and text "Mais opcoes em breve" in `#64748B`

### 5. Loading States

| Context | Loading indicator |
|---------|------------------|
| Login/signup form submission | `st.spinner("Entrando...")` or `st.spinner("Criando conta...")` wrapping the auth call |
| Page navigation | Streamlit handles natively (top progress bar) |
| Supabase data fetch | `st.spinner("Carregando...")` wrapping query calls |
| Full page load (session check) | Brief `st.spinner("Verificando sessao...")` centered on page |

### 6. Logout Flow

**Trigger:** "Sair" button in sidebar footer OR "Sair da conta" button in Settings page.

**Behavior:**
1. Call `client.auth.sign_out()`
2. Clear `st.session_state` (remove access_token, user info)
3. `st.rerun()` to redirect back to login page
4. No confirmation dialog needed (logout is low-risk, easily reversible by logging back in)

## Interaction Contracts

### Auth Flow State Machine

```
[Landing] --> session exists? --> YES --> [Dashboard]
                               --> NO  --> [Login Page]

[Login Page] --> click "Entrar" --> [Loading] --> success --> [Dashboard]
                                              --> failure --> [Login Page + Error]

[Login Page] --> click "Criar conta" link --> [Signup Mode]

[Signup Mode] --> click "Criar conta" button --> [Loading] --> success --> [Dashboard]
                                                           --> failure --> [Signup Mode + Error]

[Any Page] --> click "Sair" --> [Login Page]
```

### Session Persistence

- On successful login: store `access_token` and `user` object in `st.session_state`
- On every page load: check `st.session_state` for valid session
- If session expired: redirect to login page with no error (clean state)
- Across browser refresh: `st.session_state` persists within Streamlit session lifetime

### Navigation Guard

- If user is NOT authenticated: show only the login page, hide sidebar entirely
- If user IS authenticated: show sidebar + selected page, never show login page

## Streamlit-Specific Patterns

### Custom CSS Injection Pattern

All custom styling injected via a single `st.markdown(css, unsafe_allow_html=True)` call at the top of the app entry point. CSS is stored in a dedicated function or constant, not scattered across pages.

```python
def inject_custom_css():
    st.markdown("""
    <style>
    /* All custom overrides here */
    </style>
    """, unsafe_allow_html=True)
```

### Page Structure Pattern

```python
# app.py (entry point)
# 1. Set page config (must be first Streamlit call)
st.set_page_config(page_title="FinCoach AI", page_icon="chart_with_upwards_trend", layout="wide")

# 2. Inject custom CSS
inject_custom_css()

# 3. Check authentication
if not is_authenticated():
    show_login_page()
    st.stop()

# 4. Show sidebar navigation
selected = show_sidebar()

# 5. Route to selected page
if selected == "Dashboard":
    show_dashboard()
elif selected == "Configuracoes":
    show_settings()
```

### Dark Mode Enforcement

Streamlit respects `.streamlit/config.toml` theme settings. The dark theme defined in the Design System section above is the ONLY theme. Do not provide a light/dark toggle -- the app is dark mode only per D-03.

## Accessibility Notes

- All form inputs must have visible labels (use `st.text_input(label=...)`, never empty labels)
- Error messages use `st.error()` which includes ARIA role="alert" natively
- Success messages use `st.success()` which includes appropriate ARIA attributes
- Color contrast ratios: `#F8FAFC` on `#0F172A` = 15.4:1 (exceeds WCAG AAA 7:1)
- Color contrast ratios: `#94A3B8` on `#0F172A` = 5.6:1 (exceeds WCAG AA 4.5:1)
- Focus states: Streamlit provides native focus indicators; custom CSS must not remove them
- Tab order: Streamlit manages natively for standard components
