Accent (#2563EB) reserved for:
1. Primary CTA button: "Classificar transacoes nao classificadas"
2. Sidebar brand text "FinCoach AI" (existing, do not add new uses)

Confidence badge colors:
- High (≥0.8): #22C55E background, #F8FAFC text — no badge rendered (clean row)

Source: colors confirmed from `styles.py`. Warning yellow (#EAB308) is new for medium — but medium shows no badge (D-11), so only used as a reference.

## Component Inventory

### New: Transactions Page (`src/pages/transactions.py`)

**Page header:**
- `st.title("Transacoes")` — Streamlit default h1
- Classify button immediately below title (full-width on mobile, auto-width on desktop)

**Classify button:**
```
CSS class: .classify-btn
Background: #2563EB, color: #F8FAFC, border: none
Hover: #1D4ED8
Label: "Classificar transacoes nao classificadas"
Disabled state: when no unclassified transactions exist — button disabled + caption "Todas as transacoes ja foram classificadas"
Loading state: st.spinner("Classificando... Isso pode levar alguns segundos.")
```

**Transactions table:**
- Rendered via `st.dataframe()` with column configuration for formatting
- Columns: Data | Descricao | Merchant | Categoria | Valor | Confianca | Corrigir
- "Corrigir" column: only visible for low-confidence rows — renders `st.selectbox()` inline or via `st.data_editor()`
- Row styling: low-confidence rows get left border accent via CSS `tr.low-confidence { border-left: 3px solid #EF4444; }`
- Valor formatting: BRL currency ("R$ 1.234,56"), red text for debits (#EF4444), green for credits (#22C55E)

**Low-confidence badge (CSS class: `.confidence-badge-low`):**
```css
.confidence-badge-low {
    display: inline-flex;
    align-items: center;
    gap: 4px;
    background-color: rgba(239, 68, 68, 0.15);
    border: 1px solid #EF4444;
    border-radius: 4px;
    padding: 2px 6px;
    font-size: 12px;
    font-weight: 600;
    color: #EF4444;
}
```
Badge content: "? Baixa" — question mark + label

**Inline correction interaction (D-03):**
- Low-confidence rows show a selectbox in the "Corrigir" column
- Selectbox options: all category names (default + user-created)
- On selection change: update `category_id` + set `confidence_score = 1.0` + `manually_reviewed = true` in Supabase
- After save: row confidence badge disappears, row style resets to normal

**Empty state (no transactions at all):**
```
CSS class: .empty-state (existing)
Heading: "Nenhuma transacao encontrada"
Body: "Gere dados de teste nas Configuracoes para comecar."
```

**Empty state (all classified, no unclassified):**
- No separate empty state — classify button becomes disabled with caption (see above)

---

### Extended: Settings Page — "Categorias" section (`src/pages/settings.py`)

Inserted between "Dados de Teste" divider and "Sessao" divider.

**Section heading:** `st.header("Categorias")`

**Category list display:**
- Each category rendered as a row: [emoji] [name] [cor hex badge] [Renomear button] [Excluir button]
- Default categories: Excluir button disabled, tooltip "Categoria padrao — nao pode ser removida"
- Custom categories: both Renomear and Excluir enabled

**Add category form:**
- `st.text_input("Nome da categoria")` — max 50 chars
- `st.color_picker("Cor")` — hex color for Plotly charts
- `st.text_input("Emoji (opcional)")` — single emoji character, placeholder "🏷️"
- Submit button: "Adicionar categoria" — default Streamlit style (not accent — secondary action)

**Delete confirmation (D-06, matching existing confirm_clear pattern):**
```
st.warning("Tem certeza? Isso removera a categoria '{name}'. Transacoes associadas ficarao sem categoria.")
col1: "Confirmar exclusao" (destructive-btn class)
col2: "Cancelar"
```

**Rename inline:**
- Click "Renomear" → text_input appears in-row with current name pre-filled
- "Salvar" button confirms, "Cancelar" dismisses
- No separate modal — inline only (Streamlit pattern, no dialog component)

## Interaction States

### Classify Button States

| State | Visual | Condition |
|-------|--------|-----------|
| Default | Accent bg (#2563EB), white text | Unclassified transactions exist |
| Disabled | Streamlit disabled style, caption below | No unclassified transactions |
| Loading | st.spinner replaces button area | Webhook call in flight |
| Success | st.success toast above table | Make.com returns success |
| Error | st.error above button | httpx exception or Make.com error response |

### Inline Correction States

| State | Visual |
|-------|--------|
| Pre-correction | Low-confidence row: red left border, "? Baixa" badge, selectbox visible |
| Saving | Streamlit reruns on selectbox change (no explicit spinner needed) |
| Post-correction | Row border removed, badge removed, selected category displayed |

### Category CRUD States

| Action | State | Visual |
|--------|-------|--------|
| Add — success | st.success | "Categoria adicionada com sucesso." |
| Add — duplicate name | st.error | "Ja existe uma categoria com esse nome." |
| Add — empty name | st.error | "O nome da categoria nao pode estar vazio." |
| Delete — default category | Button disabled | Tooltip shown on hover |
| Delete — confirm flow | st.warning + two buttons | Matching existing confirm_clear pattern |
| Rename — success | st.toast | "Categoria renomeada com sucesso." |
